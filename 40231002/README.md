# 40231002 — Taxi App (Snapp-like) — Microservices Architecture

پیاده‌سازی اولیه بر اساس معماری طراحی‌شده (4+1 View Model).

## دلیل انتخاب معماری میکروسرویس
- مقیاس‌پذیری جداگانه برای هر بخش (مسافر، راننده، پرداخت)
- افزایش پایداری/مقاوم‌بودن در برابر خرابی یک سرویس
- مناسب برای پرداخت با رمزارز که نیاز به طراحی مستقل امنیتی دارد
- مناسب برای سرویس مسیریابی که از زیرساخت متفاوتی (نقشه/GPS) استفاده می‌کند

## سرویس‌ها (Logical / Development View)
| سرویس | مسئولیت |
|---|---|
| `user_service` | ثبت‌نام، ورود، احراز هویت کاربر و راننده |
| `trip_service` | ثبت درخواست سفر، انتخاب مبدا/مقصد، محاسبه کرایه |
| `routing_service` | مسیریابی چندمقصدی، تخمین مسیر/زمان |
| `payment_service` | پرداخت با رمزارز، تایید تراکنش بلاکچین |
| `wallet_service` | مدیریت کیف‌پول کاربر و راننده، واریز سهم راننده |
| `api_gateway` | درگاه واحد ورودی، مسیردهی به سرویس‌ها، احراز هویت اولیه |

ارتباط بین سرویس‌ها از طریق REST API پشت `api_gateway` و پیام‌رسانی رویداد-محور (event bus ساده) برای رویدادهایی مثل "پرداخت تکمیل شد" یا "سفر پایان یافت".

## Process View (جریان اصلی)
1. کاربر درخواست سفر ثبت می‌کند (`trip_service`)
2. `trip_service` نزدیک‌ترین راننده مناسب را پیدا می‌کند
3. `routing_service` مسیر چندمقصدی را محاسبه می‌کند
4. پس از پایان سفر، `payment_service` هزینه را با رمزارز دریافت می‌کند
5. `wallet_service` سهم راننده را واریز و در کیف‌پول او ثبت می‌کند
